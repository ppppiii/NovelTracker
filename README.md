# 📚 NovelTracker (小说进度追踪器)

> 一个轻量、优美且极其注重隐私的纯本地网文进度追踪 Web App。

## 🌟 为什么做这个项目？
很多网文作者（包括我）同时双开、三开小说时，很容易搞混哪本发了、哪本断更了、存稿还剩几天。市面上的工具要么太笨重，要么强制要求联网注册，让作者对“存稿泄露”充满担忧。

**NovelTracker** 是一次“返璞归真”的尝试：整个应用只有**一个 HTML 文件**（自带 JS/CSS）。你甚至可以把它下载到电脑桌面上双击打开运行。没有后端，没有数据库，你的灵感与数据，永远只属于你自己。

## 🚀 核心功能

*   **📈 甘特图日历看板**
    *   将冰冷的“已发字数”转化为直观的日历色块。
    *   **动态悬浮显示**：鼠标划过进度条，精准显示每一天“更新了多少章”、“是否断更”、“消耗了哪天的存稿”。
*   **🎯 存稿与发文分离管理**
    *   分别记录“今日新写”和“今日发文”。
    *   系统会自动计算你的“当前存稿量”，并在日历上用半透明斜线标出你的存稿能“撑到哪一天”。
*   **🛠️ 批量操作神器**
    *   **一键打卡**：点一下，所有【连载中】的小说自动按计划日更量发文。
    *   **全局推进/回退**：忘记打卡了？或者想撤销错误操作？利用全局时间调整，让进度精准穿越。
*   **💾 数据安全与多端迁移**
    *   全本地 `LocalStorage` 存储。
    *   支持一键导出为 `.json` 备份文件，换电脑、换浏览器随时导入恢复。
*   **🎨 丝滑的 UI 体验**
    *   支持 Light / Dark 明暗双主题无缝切换。
    *   支持任意拖拽卡片调整小说排序。

## 🛠️ 如何使用？

*   **方式一（最推荐）**：点击本仓库的 `index.html`，保存到本地，用任何浏览器打开即可。
*   **方式二（开发者在线部署）**：将本项目直接 Fork 并通过 GitHub Pages 免费开启在线访问。
